### 1.git方案
这是一份面向团队协作的 Git 版本控制与贡献操作指南。文档采用标准的开发工作流，确保每位成员的修改都能被追溯，同时保障主分支（`main`/`master`）的稳定性。
为了实现高效的版本控制、便于留下作者署名，并确保线上网站的平稳更新，本项目全面采用 Git 进行代码与内容的生命周期管理。
这份指南已为你极简压缩，重点突出，方便团队成员随时复制命令。
#### 🛠️ 核心工作流

- **核心原则**：严禁直接推送到 `main` 分支。所有修改必须通过 **独立分支** 提交 **Pull Request (PR)**。
- **基本流向**：`main(拉取最新)` ➔ `个人分支(写作/提交)` ➔ `推送远程` ➔ `PR(评审合并)`
- 推荐拉取之后使用obsidian进行编辑查看
#### 🚀 协同开发三步走

 第一步：初次拉取与配置（仅需执行一次）

Bash

```
# 克隆并进入项目
git clone https://github.com/happyADD/ratRace-lies-within.git
cd ratRace-lies-within

# 配置全局/本地署名（用于统计贡献）
git config user.name "你的名字/ID"
git config user.email "你的邮箱"
```

 第二步：后续写作日常循环（每次写新内容前）

Bash

```
# 1. 同步主分支最新代码
git checkout main
git pull origin main

# 2. 创建并切换到新分支（分支名字推荐用自己的昵称或拼音）
git checkout -b <你的分支名>

# 3. [日常循环] 本地写作并提交
git add .
git commit -m "<修改类型>(你的名字): 简短描述"
```

 第三步：推送与发起 PR（阶段性成果同步）

Bash

1. **推送分支到 GitHub（首次推送需加 -u）**
```
git push -u origin <你的分支名>
```

**2. 网页端操作**：打开 [GitHub 仓库页](https://www.google.com/search?q=https://github.com/happyADD/ratRace-lies-within&authuser=2) ➔ 点击黄色的 **"Compare & pull request"** ➔ 填写说明并点击 **"Create pull request"**。

#### 📢 团队协作规范
 Commit 信息格式
> **`修改类型： 简短描述`**
- `add: 新增第二章关于学术内卷的内容`
- `fix: 修复第一章的错别字`
主分支与部署
- `main` 分支已保护，必须通过 PR 评审后合并。
- PR 合并后，系统会自动触发 CI/CD 并更新部署线上网站。
